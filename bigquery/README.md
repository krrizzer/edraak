# Edraak BigQuery SQL

This folder contains BigQuery Standard SQL scripts for manually setting up the Edraak cross-bank data model without running Terraform.

## Tables

The manual SQL creates two datasets' tables.

Simulated bank-core tables in `bank_cores`:

- `accounts`, `transactions`, and `loans` тАФ the complete synthetic world served by the gateway
- `consents` тАФ durable append-only bank-side consent state
- `seed_meta` тАФ the date to which the demo world is anchored

Product source tables in `edraak_finance`:

- `customers` тАФ identity and employment
- `accounts` тАФ one row per bank account at every bank
- `transactions` тАФ cross-bank feed with merchant, channel, and messy
  `raw_description` strings; intentionally no trusted source category
- `loans` тАФ loans across all banks, with `remaining_months`

Derived tables (built by the backend):

- `user_profiles` тАФ cross-bank aggregates
- `detected_obligations` тАФ cache of the Transaction Intelligence Agent output
- `transaction_classifications` тАФ AI-derived transaction meaning, kept separate
  from immutable source-bank rows

Storage-only tables (agents never read these):

- `decision_requests`
- `recommendations`
- `alerts`
- `ob_consents`
- `ob_raw_payloads`

## How To Use

Open each SQL file in the BigQuery console, replace `YOUR_PROJECT_ID` with your real Google Cloud project ID, then run:

1. `01_create_dataset.sql`
2. `02_create_tables.sql`
3. `04_example_queries.sql` (optional sanity checks)

The default dataset name is `edraak_finance`.

## Seed Data

Sample data is generated by Python so dates stay relative to the current day.
The deployed backend refreshes it automatically on startup; this command is a
manual recovery option:

```bash
cd cloud-run/edrak
GCP_PROJECT_ID=your-project python -m app.data.seed.load_seed_data
```
