# Edraak BigQuery SQL

This folder contains BigQuery Standard SQL scripts for manually setting up the Edraak cross-bank data model without running Terraform.

## Tables

Source banking tables (simulated SAMA Open Banking feed):

- `customers` — identity and employment
- `accounts` — one row per bank account at every bank
- `transactions` — cross-bank feed with messy `raw_description` strings
- `loans` — loans across all banks, with `remaining_months`

Derived tables (built by the backend):

- `user_profiles` — cross-bank aggregates
- `detected_obligations` — cache of the Transaction Intelligence Agent output

Storage-only tables (agents never read these):

- `decision_requests`
- `recommendations`
- `alerts`

## How To Use

Open each SQL file in the BigQuery console, replace `YOUR_PROJECT_ID` with your real Google Cloud project ID, then run:

1. `01_create_dataset.sql`
2. `02_create_tables.sql`
3. `04_example_queries.sql` (optional sanity checks)

The default dataset name is `edraak_finance`.

## Seed Data

Sample data is generated by the Python seeder (not SQL) so dates stay relative
to the day you load it — that is what makes the Financial Radar demo work:

```bash
cd cloud-run/edrak
GCP_PROJECT_ID=your-project python -m app.data.seed.load_seed_data
```
